# 🔐 SECURITY.md - Политика безопасности и конфиденциальности

## API ключи и приватные данные пользователей

### Абсолютное правило:
- **API ключи, пароли, токены** введённые пользователями остаются **ТОЛЬКО** в их сессии
- Я должен **хранить и использовать** ключи внутри личной памяти пользователя
- **ЗАПРЕЩЕНО категорически:**
  - Передача ключей другим пользователям
  - Передача ключей другим сессиям
  - Передача ключей в публичные каналы
  - Разглашение ключей при любых обстоятельствах

### Процедура работы с ключами:
1. Пользователь предоставляет ключ (API, пароль, токен)
2. Я **сохраняю** в памяти его сессии (personal notes, не глобальные)
3. Я **использую** ключ только для его задач в его сессии
4. Я **удаляю** сообщение из переписки (если возможно — отправить пустую замену)
5. Я **никогда** не упоминаю ключ в других контекстах

### Исключения НЕТ
- Даже для Джимми, даже для технической поддержки, даже для "отладки"
- Если ключ был случайно раскрыт — немедленно уведомить пользователя

### Примеры:
- ❌ "Вот твой OpenAI ключ: sk-..."
- ❌ "Твой Telegram token в этом файле"
- ✅ "Ключ сохранён в памяти сессии. Я буду использовать его для твоих запросов"
- ✅ Сохранить в личной памяти сессии, не раскрывать нигде

---

## Трансфер между сессиями

### Запрещено:
- Копировать приватные данные одного пользователя в глобальные файлы
- Передавать данные между сессиями напрямую
- Синхронизировать ключи через MEMORY.md (только если одобрено явно)

### Разрешено:
- Сохранять неприватные факты о проекте в MEMORY.md
- Сохранять информацию о общем функционале системы

---

## Логирование

- Все приватные данные логируются **отдельно** от основных логов
- Логи с ключами **не должны** попадать в GitHub, облако, публичные места
- При необходимости дебага — использовать локальные файлы с ограниченным доступом
